# Contributing to This Project

We’re glad you’re considering a contribution. Please read these guidelines before getting started.

## Important Note

We value contributions, but **minor typo-only changes (e.g., small spelling or punctuation tweaks, or trivial rewording) will be ignored** unless they materially improve clarity or address a critical issue. If you’re unsure whether a change is substantial, open an issue first to discuss it.

# How to Contribute

## Reporting Issues

If you discover a bug or security issue, check existing issues before filing a new one.

Include as much detail as possible: steps to reproduce, expected behavior, and actual behavior.

## Proposing Code Changes

Fork the repository and create a new branch for your work. Base your branch on the latest main.

Follow the project’s coding style and conventions.

For significant changes, please open an issue to discuss the approach first.

## Submitting a Pull Request

Make sure your changes are well-tested and provide a clear summary in the pull request.

Reference related issue numbers and respond promptly to maintainer feedback.

## Code Standards

Keep to the existing structure and formatting.

Use clear, meaningful commit messages.

Verify all tests pass before submitting.
